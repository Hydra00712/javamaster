Why Comment Your Code?

Comments are non-executable statements that you add to make your code more readable and understandable. They are ignored by the compiler and are meant for human readers of your code. Proper commenting has several benefits:

Enhances Collaboration

Well-commented code makes it easier for team members to understand and work with your code.

Saves Time

When you revisit your code after a long time, comments help you quickly understand what it does and why.

Aids Debugging

Comments can help identify the purpose of a specific block of code, making it easier to find and fix bugs.

Creates Documentation

With tools like Javadoc, comments can be used to generate comprehensive code documentation.

Types of Comments in Java

Java supports three types of comments:

1. Single-line Comments

Single-line comments start with // and continue until the end of the line.

Java 
// This is a single-line comment
int age = 25; // This comment is on the same line as code

2. Multi-line Comments

Multi-line comments start with /* and end with */. Everything between these markers is treated as a comment, even if it spans multiple lines.

Java 
/* This is a
   multi-line
   comment */
   
/* This is another way to write
 * a multi-line comment, with asterisks
 * at the beginning of each line
 */

3. Documentation Comments (Javadoc)

Documentation comments start with /** and end with */. They are used to generate documentation using the Javadoc tool.

Java 
/**
 * This is a Javadoc comment
 * It is used to describe classes, methods, and fields
 * 
 * @author John Doe
 * @version 1.0
 */
public class Calculator {
    // Class implementation
}

Javadoc Documentation

Javadoc is a documentation generator for Java that generates HTML documentation from Java source code, based on the documentation comments. Here's how to use it effectively:

Javadoc Tags

Javadoc comments support special tags that start with @. Here are some commonly used tags:

Tag Purpose Example
@param Documents a method parameter @param name The name of the user
@return Documents the return value @return The sum of two numbers
@throws Documents an exception that may be thrown @throws IOException If the file cannot be read
@author Specifies the author of the class @author John Doe
@version Indicates the version of the class @version 1.0
@see Provides a link to another class or method @see java.util.ArrayList
@since Specifies when this feature was added @since 1.2
@deprecated Indicates that this feature is deprecated @deprecated Use newMethod() instead

Generating Javadoc

To generate Javadoc documentation, you can use the javadoc command-line tool or your IDE's built-in Javadoc generator:

Command 
javadoc -d docs src/*.java

This command generates documentation for all Java files in the src directory and saves it in the docs directory.

Note

Most modern IDEs like IntelliJ IDEA and Eclipse provide convenient ways to generate Javadoc. For example, in IntelliJ IDEA, you can go to Tools > Generate JavaDoc.

Best Practices for Commenting

Follow these guidelines to write effective comments:

Write Clear and Concise Comments

Comments should be clear, concise, and add value. Avoid stating the obvious.

Poor Example:

// Increment i by 1
i++;

Better Example:

// Move to the next student in the array
i++;

Comment Why, Not What

Good comments explain why something is done, not what is being done (the code itself shows what is being done).

Poor Example:

// Setting count to zero
count = 0;

Better Example:

// Reset counter before processing new batch
count = 0;

Document Public APIs with Javadoc

All public classes, methods, and fields should be documented with Javadoc comments, especially if your code will be used by other developers.

Keep Comments Updated

When you change code, update the comments to reflect those changes. Outdated comments are misleading and can cause confusion.

Use Comments for TODOs

Use // TODO: comments to mark code that needs further attention or improvement. Most IDEs recognize this format and can track these items.

// TODO: Implement input validation here

Comprehensive Examples

Let's look at some comprehensive examples of well-commented Java code:

Example 1: A Well-Documented Class

Java 
/**
 * A simple calculator class that provides basic arithmetic operations.
 * 
 * This class demonstrates proper documentation using Javadoc comments.
 * 
 * @author John Doe
 * @version 1.0
 * @since 2023-01-01
 */
public class Calculator {
    
    /**
     * Adds two integers and returns the result.
     * 
     * @param a The first integer to add
     * @param b The second integer to add
     * @return The sum of the two integers
     */
    public int add(int a, int b) {
        return a + b;
    }
    
    /**
     * Subtracts the second integer from the first and returns the result.
     * 
     * @param a The integer to subtract from
     * @param b The integer to subtract
     * @return The difference between a and b
     */
    public int subtract(int a, int b) {
        return a - b;
    }
    
    /**
     * Divides the first integer by the second and returns the result.
     * 
     * @param a The dividend
     * @param b The divisor
     * @return The quotient of a divided by b
     * @throws ArithmeticException If the divisor is zero
     */
    public double divide(int a, int b) {
        // Check for division by zero
        if (b == 0) {
            throw new ArithmeticException("Cannot divide by zero");
        }
        
        return (double) a / b;
    }
}

Example 2: Using Comments to Explain Complex Logic

Java 
public class SortingExample {
    
    /**
     * Implements the quicksort algorithm to sort an array.
     * 
     * @param arr The array to be sorted
     * @param low The starting index
     * @param high The ending index
     */
    public void quickSort(int[] arr, int low, int high) {
        if (low < high) {
            /* 
             * Partitioning: This is the key step in quicksort.
             * We select a pivot element and place it in its correct position
             * such that all elements less than the pivot are to the left
             * and all elements greater than the pivot are to the right.
             */
            int pivotIndex = partition(arr, low, high);
            
            // Recursively sort elements before and after the pivot
            quickSort(arr, low, pivotIndex - 1);
            quickSort(arr, pivotIndex + 1, high);
        }
    }
    
    /**
     * Helper method for quickSort that partitions the array.
     * 
     * @param arr The array to be partitioned
     * @param low The starting index
     * @param high The ending index (pivot element)
     * @return The index of the pivot element after partitioning
     */
    private int partition(int[] arr, int low, int high) {
        // We use the rightmost element as the pivot
        int pivot = arr[high];
        
        // Index of smaller element
        int i = (low - 1);
        
        // Move through the array and place elements
        // smaller than the pivot to the left
        for (int j = low; j < high; j++) {
            if (arr[j] <= pivot) {
                i++;
                
                // Swap elements
                int temp = arr[i];
                arr[i] = arr[j];
                arr[j] = temp;
            }
        }
        
        // Place the pivot element in its correct position
        int temp = arr[i + 1];
        arr[i + 1] = arr[high];
        arr[high] = temp;
        
        return i + 1;
    }
}

Key Takeaways

  • Comments are essential for making your code readable and maintainable.
  • Java supports three types of comments: single-line (//), multi-line (/* */), and Javadoc (/** */).
  • Javadoc comments can be processed by the Javadoc tool to generate HTML documentation.
  • Good comments explain why something is done, not what is being done.
  • Keep comments updated when you change code to avoid confusion.
  • Use // TODO: comments to mark areas that need future improvement.
